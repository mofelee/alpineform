## Summary

- First complete AlpineForm alpha: a reviewable validate, plan, apply, and check
  loop for persistent Alpine 3.24 hosts.
- Real Alpine 3.24.1 x86_64 VM gates cover the complete Beta core.
- Four static CLI archives ship with checksums, SBOMs, Sigstore verification,
  GitHub provenance, and a checksum-verifying installer.

## Compatibility

- Release phase: alpha; stable compatibility is not promised.
- CLI artifacts: Linux and macOS on amd64 and arm64.
- Beta managed target: Alpine 3.24 x86_64.
- Preview managed target: Alpine 3.24 aarch64; no blocking real-VM gate.
- This release supersedes the incomplete alpha.1 through alpha.3 prereleases.
  There is no AlpineForm state migration.

## Breaking Changes

- None relative to the alpha.3 code or state format.

## Migration Notes

- Do not use alpha.1 through alpha.3 release artifacts. Install alpha.4 directly.
- Back up `/var/lib/alpineform/state.json` before later alpha upgrades.

## Added

- Alpine fact discovery, root SSH, atomic state, renewable leases, deterministic
  plans, native Alpine resources, verified prebuilt components, and deduplicated
  change scripts.
- Blocking no-op, drift, repair, lifecycle, secret, and reboot VM assertions.
- Signed release checksums, per-archive SPDX SBOMs, GitHub attestations, and the
  checksum-verifying installer.

## Changed

- Release eligibility is checked before snapshot construction or publication.
- Verification results are recursively discovered and parsed as a fixed set of
  allowed records before the release matrix is published.

## Fixed

- Install Cosign under the command name invoked by the release workflow.
- Aggregate nested Linux, macOS, supply-chain, and fresh-Alpine result artifacts
  without executing artifact-provided shell content.
- Retain failed prereleases without moving tags or replacing partial assets,
  then publish corrections under new versions.

## Security

- Protected values are omitted from plan, graph, state, HTML, debug, errors, and
  uploaded integration diagnostics.
- Component artifacts and release archives are checksum verified before use.

## Known Issues

- Alpine 3.24 aarch64 target support is Preview because it lacks a real-VM gate.
- Custom APK signing keys and CA-certificate components have unit/provider
  coverage but are not in the blocking VM fixture.
- Homebrew is not published for this alpha.
- Docker (#10), rollback-safe nftables (#13), and target-side source builds
  (#14) remain independent follow-ups outside the v0.1 core.

## Verification

- The tag target must have successful `CI` and `Release dry-run` checks.
- The release workflow verifies assets, checksums, SBOMs, Sigstore bundle,
  provenance, installer metadata, and the promoted quickstart on a fresh Alpine
  3.24.1 VM before writing the final matrix below.
